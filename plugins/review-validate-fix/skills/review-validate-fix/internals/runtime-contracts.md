# Runtime Contracts

Audience: RVF maintainers. Normal RVF agents should not read this file.

## Single Sources Of Truth

- Backend selection: `scripts/codex_stop_review_validate_fix.py`
- Router channel selection: `scripts/codex_stop_hook_router.py`
- Dispatcher dev/stable handoff: `scripts/codex_stop_hook_dispatcher.py`
- Kanban CLI integration: `scripts/cline_kanban_client.py`
- Run ledger fields: `scripts/rvf_logging.py`
- Review prep artifacts: `scripts/prepare_review_run.py`
- Reviewer protocol: `scripts/write_review_result.py` and `scripts/check_review_result.py`
- Handoff behavior: `scripts/rvf_handoff.py`

## Documentation Rule

Do not duplicate env var tables, state machines, or summary schemas in `SKILL.md`. If maintainers need a readable explanation, keep it here and point to the owning script. If agents need to recover from a failure, provide a short runbook in `debug/`.

## Agent Visibility

Runtime scripts may make env vars automatically visible to subprocesses or generated prompts. Agents should use generated context files when they are present, but they should not hand-craft env/config payloads for scripts.

Subagents generated by RVF receive self-contained prompt artifacts from `prompts/` plus run artifacts. They do not need to access `SKILL.md`.
